# Plan: Remove Chat History + Add Interrupt Capability

## Summary
Remove the chat message bubbles/history display and only show streaming text. Add interrupt capability so users can press push-to-talk at any time to stop AI and listen to user.

## Goals
1. Remove conversation history display (ConversationHistory component)
2. Only show the current streaming message from AI
3. Allow interrupting AI speech at any time by pressing push-to-talk
4. When interrupted: stop audio playback, clear current stream, and start recording immediately

## Implementation Details

### Interrupt Flow:
1. User presses push-to-talk button while AI is speaking
2. `handlePressStart` detects `isStreaming` or `isProcessing` is true
3. Calls `stopSpeaking()` which:
   - Cancels the fetch request to `/api/chat-stream`
   - Clears audio queue
   - Stops any currently playing audio
   - Resets state: `isStreaming=false`, `isProcessing=false`
   - Clears `currentStreamingText`
4. Immediately starts recording user audio
5. UI only shows current streaming text (no history)

### State Flow After Changes:
- Idle → Recording → Processing/Streaming → Idle
- Any state → Recording (interrupt): Cancel current operations, reset, start recording
